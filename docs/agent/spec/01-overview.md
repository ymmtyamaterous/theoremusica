# Teoria — 全体概要仕様書

> 作成日: 2026-03-22

---

## 1. プロジェクト概要

**Teoria** は、音楽理論を体系的かつインタラクティブに学習できる Web プラットフォームです。  
初心者から上級者まで、楽譜の読み方・スケール・和声・対位法・ジャズ理論を自分のペースで習得できる環境を提供します。

---

---

## 3. 主要機能一覧

### 3.1 認証・ユーザー管理
- メールアドレス / パスワードによる会員登録・ログイン
- セッション管理（better-auth）
- ユーザープロフィール表示

### 3.2 コース管理
- コース一覧の表示（レベル・カテゴリでフィルタ）
- コース詳細（概要・レッスン一覧・所要時間）
- コースへの受講登録 (enrollment)

### 3.3 レッスン学習
- テキストベースのレッスンコンテンツ表示
- レッスン完了マーク機能
- 前後レッスンへのナビゲーション

### 3.4 インタラクティブ学習ツール
- **ピアノキーボード**：音符をクリックして音を視覚的に確認（ヒーロー表示用＋学習ツール用）
- **五度圏 (Circle of Fifths)**：インタラクティブな SVG で調性の関係を探索

### 3.5 学習進捗管理
- コース単位・レッスン単位の進捗表示
- ダッシュボードで全体の学習状況を確認
- 完了済みレッスン数・受講中コース数のサマリー

## 4. 画面一覧

| パス | 画面名 | 認証要否 |
|------|--------|---------|
| `/` | トップ (LP) | 不要 |
| `/login` | ログイン | 不要 |
| `/register` | 新規登録 | 不要 |
| `/courses` | コース一覧 | 不要 |
| `/courses/:courseId` | コース詳細 | 不要 |
| `/courses/:courseId/lessons/:lessonId` | レッスン | 要 |
| `/dashboard` | ダッシュボード | 要 |

---

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンシブデザイン | モバイル・タブレット・デスクトップ対応 |
| デザインテーマ | ダーク背景 (`#0e0d0c`)・ゴールドアクセント (`#c9a84c`) を基調とした音楽系デザイン |
| フォント | Playfair Display（見出し）/ DM Sans（本文） |
| アクセシビリティ | WCAG 2.1 AA 準拠を目指す |
| セキュリティ | 認証必須ルートは適切にガード・セッションベース認証 |

## 6. コース定義

| コースID | タイトル | レベル | レッスン数 | 推定時間 |
|----------|---------|--------|-----------|---------|
| 1 | 楽譜の読み方 | 入門 | 12 | 約 4 時間 |
| 2 | スケールと調性 | 中級 | 18 | 約 7 時間 |
| 3 | 和音と和声進行 | 中級 | 22 | 約 9 時間 |
| 4 | 対位法 | 上級 | 30 | 約 14 時間 |
| 5 | リズムと拍子 | 入門 | 10 | 約 3 時間 |
| 6 | ジャズ和声 | 上級 | 28 | 約 12 時間 |

## 7. ディレクトリ構成 (主要部分)

```
better-t-app/
├── apps/
│   ├── web/         # React フロントエンド
│   └── server/      # Hono バックエンド
└── packages/
    ├── api/         # oRPC ルーター定義
    ├── auth/        # better-auth 設定
    ├── db/          # Drizzle スキーマ・マイグレーション
    ├── env/         # 環境変数バリデーション
    ├── ui/          # 共通 UI コンポーネント
    └── config/      # 共通 tsconfig
```
